# SEO audit — 2026-05-23

Scope: all public pages — `/`, `/vehicules`, `/vehicules/[slug]`, `/blog`, `/blog/[slug]`, `/contact`.

Assumed canonical domain: **https://klavem.fr** (matches `contact@klavem.fr`).

## Critical (search engines + social previews break or look wrong)

- [x] **Root layout has Create-Next-App placeholder metadata.**
  `src/app/layout.tsx` exports `title: "Create Next App"` / `description: "Generated by create next app"`. This leaks through to any route that doesn't override (none currently override openGraph/twitter, so social shares show the wrong title).
- [x] **`<html lang="en">` but the site is fully in French.** Hurts language detection and accessibility tooling.
- [x] **No `metadataBase`.** Without it, OG image URLs and canonicals stay relative and crawlers can't resolve them.
- [x] **No `robots.txt`.** Need `app/robots.ts` (or static `public/robots.txt`).
- [x] **No `sitemap.xml`.** Need `app/sitemap.ts` covering home, vehicle list, each vehicle, blog list, each blog post, contact.
- [x] **No `openGraph` / `twitter` metadata anywhere.** Shared links on FB, LinkedIn, WhatsApp, Slack will show a default URL preview without title/image.
- [ ] **No Open Graph image** (1200×630). At minimum a static `app/opengraph-image.png` so every share has the Klavem brand visible.

## Important (rich results + crawl guidance)

- [x] **`/vehicules` page has no `<h1>`** — only a `<h2>` ("Choisissez votre véhicule"). Every indexable page should have exactly one `<h1>`.
- [x] **No `Organization` / `LocalBusiness` JSON-LD.** Klavem is a local service (Île-de-France only) — perfect fit for `LocalBusiness` schema with address, phone, opening hours.
- [x] **No `Vehicle` / `Product` JSON-LD on vehicle pages.** Vehicle detail pages should surface make/model/price as structured data so Google can show vehicle rich results.
- [x] **No `BlogPosting` JSON-LD on blog articles.** Already have headline + excerpt — easy to wire.
- [ ] **No `BreadcrumbList` JSON-LD** on inner pages (vehicle detail, blog post).
- [x] **No canonical URLs** declared via `alternates.canonical`. With duplicate-content risk from `?color=` and other query params, canonicals are important.

## Medium

- [x] **Description copy unfocused** on home and contact — could lead with the value prop instead of summarising. Current home description is decent but could push the price point earlier.
- [x] **Empty `alt=""` on hero and zone-section images.** Decorative images use empty alt by convention — flag for review but currently acceptable.
- [ ] **Vehicle card alt text** already includes brand + "VTC Île-de-France" — good for SEO. No fix needed.
- [ ] **Testimonial alt is just the person's name** — minor; "Témoignage de {name}" is slightly better but not critical.
- [ ] **No `apple-mobile-web-app-*`** or manifest metadata. Low impact on SEO; primarily a PWA concern.

## Existing strengths

- FAQ section already emits `FAQPage` JSON-LD (`src/components/sections/faq-section.tsx`). Good.
- Vehicle detail page already calls `generateMetadata` with model-specific title + description. Good base.
- Static generation via `generateStaticParams` covers all vehicles and blog posts at build time. Good for crawl efficiency.
- App router favicon convention used (`apple-icon.png`, `icon.png`, `icon1.png`, `icon2.png`). Good.

## Fix plan (executed in this pass)

1. **Root layout** — replace placeholder metadata with brand metadata, set `lang="fr"`, set `metadataBase`, add `title.template` so every page becomes `… | Klavem Fleet`, default `openGraph` and `twitter` cards.
2. **`app/robots.ts`** — allow all, point to sitemap.
3. **`app/sitemap.ts`** — generate URLs for home, contact, vehicle list + each vehicle, blog list + each blog post, with lastmod and priority.
4. **`Organization` / `LocalBusiness` JSON-LD** — inject in root layout (so it appears on every page).
5. **`Vehicle` JSON-LD** — add on `/vehicules/[slug]` with brand, model, year, motorisation, price.
6. **`BlogPosting` JSON-LD** — add on `/blog/[slug]` with headline, datePublished, author, image.
7. **`<h1>` on `/vehicules`** — make `ContentBlock` render the page headline at level 1.
8. **Per-route `alternates.canonical`** — set canonical URLs on each page.

## Out of scope for this pass (logged for later)

- OG image generation (text-on-image at 1200×630). Could be done with `app/opengraph-image.tsx` using OG image generation, but needs design.
- LocalBusiness `aggregateRating` / review schema (depends on whether testimonial source data is reviewable).
- Blog `Article.author` enrichment (current data has no author field — assumes "Klavem Fleet").
- Hreflang (not relevant — site is single-language).
- Sitemap image extensions (could list each vehicle image; not required).
